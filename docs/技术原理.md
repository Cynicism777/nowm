# 小红书图文笔记无水印下载

从小红书分享链接解析并下载无水印原图。核心思路：水印不是烧进原图的，而是 CDN 在展示/下载时通过图片处理模板临时叠加；只要拿到图片的 `fileId`，换用源站 CDN 域名即可获取原始上传件。

## 为什么 App 下载有水印，第三方工具没有？

小红书对同一张图片维护了两套 CDN 地址：

| CDN 域名 | 用途 | 特点 |
|---|---|---|
| `sns-webpic-*.xhscdn.com` | App / 网页展示、用户点「保存图片」 | 降分辨率（通常 1080 宽）+ 叠加水印，URL 末尾带 `!nd_dft_wlteh_jpg_3` 等处理指令 |
| `sns-img-*.xhscdn.com` | 图片源站（存用户上传的原始文件） | 原始分辨率、无平台水印 |

两套地址共用同一个 `fileId`。所谓「无水印解析」，本质就是：**从笔记数据里提取 `fileId`，拼出源站 URL，绕开加水印的处理后缀**。

实测对比（同一笔记第一张图）：

- 带水印：`sns-webpic-qc.../...!nd_dft_wlteh_jpg_3` → 1080×1440 JPEG，约 212 KB
- 无水印：`sns-img-qc.xhscdn.com/notes_pre_post/{fileId}` → 1440×1920 PNG，约 878 KB

## 技术路线

```
分享文案 / 短链
      │
      ▼
① 还原真实笔记 URL（xhslink.com → 302 → 带 xsec_token 的长链）
      │
      ▼
② 请求笔记详情页 HTML（需带 xsec_token + 浏览器 UA）
      │
      ▼
③ 从 window.__INITIAL_STATE__ 中解析 imageList[].fileId
      │
      ▼
④ 拼接源站 CDN URL，下载无水印原图
```

### 步骤 1：还原真实链接

用户从小红书复制的分享文案通常包含短链，例如：

```
http://xhslink.com/o/4bDk4vyM9uJ
```

对短链发 HTTP 请求，服务端返回 `302`，`Location` 头里是真正的笔记地址：

```
https://www.xiaohongshu.com/discovery/item/{note_id}?xsec_token=...&xsec_source=app_share&...
```

其中 `xsec_token` 是访问详情页的鉴权参数，**必须从当前分享链接实时获取，不能硬编码**。

### 步骤 2：抓取详情页

用浏览器 User-Agent 请求上一步得到的完整 URL，页面 HTML 中包含一段内嵌 JSON：

```html
<script>
  window.__INITIAL_STATE__ = { ... }
</script>
```

这是小红书 Web 端 SSR 注入的初始状态，笔记的图片、标题、作者等信息都在里面。

### 步骤 3：解析 fileId

在 `__INITIAL_STATE__` 中递归查找 `imageList` 数组，每个元素结构大致如下：

```json
{
  "fileId": "notes_pre_post/1040g3k8322hd1oaqnu704a126711jo36dfpv3lg",
  "width": 1440,
  "height": 1920,
  "urlDefault": "http://sns-webpic-qc.xhscdn.com/.../...!nd_dft_wlteh_jpg_3",
  "infoList": [
    { "imageScene": "WB_PRV", "url": "...!nd_prv_wlteh_jpg_3" },
    { "imageScene": "WB_DFT", "url": "...!nd_dft_wlteh_jpg_3" }
  ]
}
```

关键字段是 `fileId`，格式为 `{类型前缀}/{唯一标识}`，例如 `notes_pre_post/1040g3k8...`。

`urlDefault` 和 `infoList` 里的 URL 都是 `sns-webpic-*` 域名，带水印，**不要用这些**。

### 步骤 4：拼接无水印 URL

用 `fileId` 拼出源站地址：

```
https://sns-img-qc.xhscdn.com/{fileId}?imageView2/format/png
```

可用的源站域名（互为容灾）：

- `sns-img-qc.xhscdn.com`
- `sns-img-bd.xhscdn.com`
- `sns-img-hw.xhscdn.com`
- `sns-img-qn.xhscdn.com`
- `ci.xiaohongshu.com`

图片源站本身几乎不设防，直接 HTTP GET 即可下载。

## 后端实现映射

后端将上述四步拆分为职责单一的模块：

| 模块 | 对应步骤 | 作用 |
|---|---|---|
| `parser.py` | 步骤 1–3 | 从分享文案提取链接、短链 302 还原、带 `xsec_token` 抓详情页、正则提取 `__INITIAL_STATE__` 并递归找 `imageList`、收集去重 `fileId` |
| `cdn.py` | 步骤 4 | 用 `fileId` 拼源站 CDN URL，多源站容灾拉取（不带 Referer 绕过防盗链） |

核心拼接逻辑：

```python
def nowm_url(file_id: str) -> str:
    return f"https://sns-img-qc.xhscdn.com/{file_id}?imageView2/format/png"
```

服务的启动与使用方式见项目根目录 `README.md`。

## 视频笔记

图文笔记走 `imageList`，视频笔记的数据结构不同。视频的无水印直链在：

```
note.video.media.stream.h264[].masterUrl
```

同样在 `__INITIAL_STATE__` 里，解析思路一致。

## 限制与注意事项

- **xsec_token 会过期**：分享链接里的 token 有时效，过期后详情页会返回登录墙，需要重新获取分享链接。
- **反爬**：高频请求详情页可能触发验证码。稳定方案需要加请求间隔、携带 Cookie，或走已鉴权的 API。瓶颈在「抓详情页」这一步，图片源站本身宽松。
- **仅去除平台水印**：不去除原作者画在图上的水印。下载内容版权归原作者，自用可以，转载/商用需授权。
- **接口可能变化**：CDN 域名、`__INITIAL_STATE__` 结构、鉴权方式都可能随小红书版本更新而调整，需要持续关注。
